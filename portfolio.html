about

Who I am

I'm a self-directed systems programmer with a strong foundation in mathematics and computer science. My work lives at the intersection of low-level software and open-source culture — I care about how things actually work, not just what they produce.

I run NixOS as my daily driver, not because it's trendy, but because reproducibility and declarative configuration are the right way to think about systems. Most of my projects involve either modifying language runtimes or building tooling that bridges the gap between high-level languages and the OS underneath.

Long-term I want to build the kind of open-source impact that earns a place on people's git clone lists — software that outlasts any single employer.

Things I've built

A mix of experiments and serious tools. Most are about understanding what's underneath, or filling a gap that bothered me.

Ruby · C · Qt
ruby-is-qt
A handmade Qt binding library for Ruby. Not a wrapper around someone else's FFI — built from scratch to explore how GUI frameworks interact with scripting runtimes at the object model level.
GitHub → README →
C · CPython internals
cpython-no-gc
A fork of CPython with the cycle-collecting garbage collector removed. An experiment in understanding what Python's GC actually does — and what breaks when you take it away.
GitHub → Write-up →
Nix · Shell
nixos-config
My personal NixOS flake: Niri compositor, greetd/regreet display manager, Wayland stack, and all the small glue that makes a daily-driver system reproducible from scratch.
